# Support

Use GitHub issues for reproducible bugs and feature proposals. Include the Upstand version, deployment mode, relevant logs with secrets removed, exact steps, expected behavior, and actual behavior. For UI issues include the browser and a screenshot when useful.

Use discussions or documentation questions for configuration and general usage. Do not post passwords, access tokens, SSH private keys, database URLs, `.env` files, or public server addresses containing sensitive information.

Security vulnerabilities must be reported privately; see [SECURITY.md](SECURITY.md). Upstand is maintained on a best-effort basis and does not promise production incident response through public issues. For a live outage, first consult the [troubleshooting guide](apps/fumadocs/content/docs/troubleshooting.mdx), inspect `docker stack services upstand`, and preserve logs before restarting services.
