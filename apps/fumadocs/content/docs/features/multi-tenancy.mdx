Upstand is built from the ground up to support secure multi-tenant operations. Whether you are hosting services for multiple clients, managing development stages across distinct environments, or partitioning departments within an enterprise, Upstand provides clean logical boundaries.

---

## The Organization Boundary

The core of Upstand's multi-tenancy is the **Organization**. All infrastructure components and credentials belong strictly to a parent organization:

- **Servers & Nodes**: Docker Swarm managers, workers, and build servers are registered under a specific organization.
- **Resources**: Applications, compose stacks, databases, and static sites.
- **Routing**: Domains, Caddy configuration snippets, and TLS certificates.
- **Integrations**: Git providers (GitHub, GitLab), S3 storage destinations, and external Docker registries.

Users are members of one or more organizations and can seamlessly switch between active contexts from the dashboard interface.

---

## Role-Based Access Control (RBAC)

Access within an organization is governed by memberships and permissions. Upstand supports standard roles and custom permission models:

- **Owner**: Full administrative access over the organization, billing, and member lists. Can delete the organization.
- **Admin**: Can manage servers, deploy resources, invite members, and modify routing rules. Cannot delete the organization or modify other administrators.
- **Member**: Read-only dashboard access with deployment permissions constrained by specific project/environment scopes.
- **Custom Roles**: Organizations can define fine-grained custom roles targeting specific features (for example, a `Database Operator` or `Build Engineer`) by combining individual permissions:
  - `server:create`, `server:update`, `server:view`
  - `resource:deploy`, `resource:delete`, `resource:view`
  - `billing:manage`

## Member Invitations & Lifecycle

Inviting team members is handled securely via the dashboard:

1. **Invitation Creation**: An admin enters the user's email, selects their initial role, and sets their scopes.
2. **Secure Token**: Upstand generates a unique invitation record in the database with a cryptographically secure token and an expiry window (default is 7 days).
3. **Redemption**: The invitee receives the invitation link, signs up or authenticates, and is dynamically bound to the organization.

## Directory Sync (SCIM Integration)

For enterprise self-hosters, Upstand supports identity provider (IdP) directory synchronization using the **SCIM** (System for Cross-domain Identity Management) standard:

- **Automated Provisioning**: Synchronize users and groups automatically from Okta, Microsoft Entra ID (Azure AD), Google Workspace, or Ping Identity.
- **De-provisioning**: When a member is disabled or removed from your central directory, their SCIM state is marked inactive in Upstand, instantly revoking all active sessions.
- **External Identity Mapping**: Tracks IdP external identifiers (`scim_external_id`) and displays name mappings on the member records.
