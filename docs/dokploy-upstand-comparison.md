## Executive conclusion

Dokploy is currently the broader, more mature product surface. Its core advantage is depth around service-specific deployment UX and operational controls: distinct Application, Compose, and database workflows; Traefik/domain/certificate management; remote/build servers; previews, rollbacks, patches, schedules, volume backups; Docker inspection; and a 450-path generated API. Upstand has a cleaner generic-resource/domain architecture and several strong foundations—Swarm, queued deployments, encrypted secrets, S3 backup schedules and restore, provider integrations, notifications, audit logs, API keys/MCP, and UpGal—but many Dokploy capabilities are either not yet exposed, only represented as generic JSON, or absent.

This is a source and documentation audit, not a live deployment test. No external credentials, Docker cluster, Git webhook, DNS provider, or billing account was available, so runtime behavior and failure modes were not experimentally verified.

## Recommended parity order for Upstand

1. Keep resource parity production-grade: deepen dedicated application/Compose/database settings, provider-specific import behavior, domain/certificate middleware, and service-specific operational controls.
2. Keep deployment lifecycle parity production-grade: exercise every provider webhook, build-server/concurrency path, preview limit/domain path, rollback history, zero-downtime behavior, and cleanup action against live Swarm targets.
3. Close the remaining operations gaps: Traefik-equivalent documentation for the Caddy architecture, richer service terminal/log/status controls, and marketplace-scale template federation.
4. Harden infrastructure parity: remote Docker/build servers, cluster placement, GPU setup, installer/update observability, and operational permission/audit coverage.
5. Keep explicit product boundaries for libSQL, CDN, forward auth, billing, licensing, and cloud provisioning. SSO/SCIM/custom roles/whitelabeling now have organization-scoped foundations; hosted cloud provisioning remains outside the self-hosted control plane.
